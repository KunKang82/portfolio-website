Case Study for MyFlix Full-Stack Project

Overview

The myFlix project aims to develop a full-stack web application that provides users with access to information about movies, directors, and genres. The application consists of both server-side and client-side components, allowing users to interact with the system via RESTful API endpoints and a responsive user interface built with React.js.

Purpose & Objective

The purpose of the myFlix project is to create a comprehensive movie application that meets the information needs of movie enthusiasts. The objective is to build a robust backend API for data management and implement a user-friendly frontend interface for seamless navigation and interaction.

Server-side

A RESTful API was developed using Node.js and Express, communicating with a MongoDB database. The API supports standard HTTP methods and implements CRUD operations for data retrieval and storage. Postman was used for API testing, with implemented user authentication mechanisms including basic HTTP authentication and JWT authentication.

Client-side

Following API development, the client-side interface was constructed using React and React-Redux. It offers various views including movie lists, single movie details, login, registration, and profile management.

Challenges

This project presented both my favorite and most challenging experiences. I enjoyed building the API and mastering database structures, as well as becoming proficient with the terminal. However, developing the client-side posed challenges, requiring time to grasp React and Redux concepts and achieve desired outcomes. With guidance from my tutor and a pair-programming session with an experienced developer, I successfully overcame these obstacles.

Tech Stack / Tools

Backend:

  • Node.js
  • Express
  • Mongoose
  • MongoDB
  • Passport
  • JSON Web Tokens(JWT) for authentication
  • Heroku for deployment

Frontend:

  • React.js
  • React Redux for state management
  • React Router for routing
  • TypeScript
  • React Bootstrap for styling and responsiveness
  • Parcel as the build tool
  • ES2015+ for JavasScript syntax

Duration

The project spanned approximately 2 months, with 1 month dedicated to backend development and another month to frontend development.

Conclusion

The MyFlix project served as a comprehensive learning experience in full-stack web development. Traversing from server-side API development to client-side interface construction provided valuable insights into the intricacies of modern web applications. Despite the challenges encountered, the project culminated in a successful demonstration of proficiency in both backend and frontend technologies.

Possible Further Improvements

  • Enhanced error handling and validation on the server-side
  • Integration of additional third-party APIs for enriched movie data
  • Implementation of user feedback mechanisms and user ratings
  • Enhancement of user interface with more interactive elements and animations
